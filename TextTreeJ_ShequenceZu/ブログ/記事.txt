・仕事メモ
・Gradleのタスクについて

タスクの書き方は大別して二種類あってそれぞれ動作の仕方が違う

例

task MessageA {
  println("MessageA")
}

task MessageB << {
  println("MessageB")
}

上のCopyAの方は、明示的に呼ばなくても、同じgradleスクリプト内の関数の
何かが呼ばれれば、起動時に必ず呼ばれる。

つまり上の例でいうとCopyBを明示的に読んだはずなのにCopyAの方が早く呼ばれる。
この　<<　が無しの奴が複数ある場合、上から順に実行される。

一方下の << がついてる奴は、明示的に呼ばないと実行されることはない。実行順は
<< 無しより遅い。
